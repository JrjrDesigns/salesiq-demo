Action Workflows

Triggered by form submissions — separate from scheduled pipeline

Page 2 of 2
Workflow 2 — New Client  ·  POST /new-client
→
Webhook Node (POST)
Receives new client form submission
→
Validation Code Node
Required fields present? Account name, owner, territory
→
AI Mapping Code Node
Maps SF fields → Epicor schema · Generates CustomerID · Sets type, segment, address
→
Parallel API Calls
POST Salesforce Account API · POST Epicor Customer API (simultaneously)
→
Atomicity Check Code Node
Both succeeded → write to DB · Either failed → log error, alert, rollback
→
Respond to Webhook Node
Returns success or error to form · Dashboard updates within 5s
Workflow 3 — New RFQ  ·  POST /new-rfq
→
Webhook Node (POST)
Receives RFQ form submission
→
Validation Code Node
Required fields present? Company, contact, product interest
→
AI Customer Match Node
Query DB for existing customer match · Match on name, email domain, EIN · Return confidence score
→
Routing Code Node
>80% match → fast path, pre-populate fields, create SF inquiry linked to customer · No match → create new prospect inquiry, flag for qualification
→
Alert Node
Notify sales rep · Fast path or new prospect — different message
→
Respond to Webhook Node
Returns confirmation to form
Workflow 4 — Close Deal  ·  POST /close-deal
→
Webhook Node (POST)
Receives close deal form with opportunity ID
→
Validation Code Node
Opportunity exists? All required fields present?
→
PATCH Salesforce API
Update opportunity stage → Closed Won
→
AI Derivation Code Node
Map products → Epicor part numbers · Set ship to from billing address · Calculate need-by date · Set payment terms
→
POST Epicor Sales Order API
Create real Epicor order with derived fields
→
PATCH Salesforce API
Write Epicor order # back to SF opportunity · Confirms sync complete
→
Alert + Respond Node
Alert ops: new order ready for review · Return confirmation to form
Workflow 5 — Data Server  ·  GET /data
→
Webhook Node (GET)
Dashboard requests current data
→
Postgres Query Node
Read latest data from all tables · Return as unified JSON object
→
Respond to Webhook Node
Return JSON to dashboard · Dashboard re-renders with live data
Note: In full production the dashboard connects directly to PostgreSQL and this workflow is not needed. Used during transition from proof of concept.
